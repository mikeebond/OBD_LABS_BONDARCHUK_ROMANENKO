# Лабораторна робота 4: Аналітичні SQL-запити (OLAP)
**Тема проєкту:** Вебзастосунок для ведення особистого бюджету.

У цьому файлі наведено опис аналітичних запитів, створених для аналізу даних користувачів, їхніх фінансів та транзакцій.

## 1. Агрегатні функції та групування (GROUP BY, HAVING)
Ці запити використовуються для підбиття підсумків та зведеної статистики:
* **Запит 1.1:** Використовує `COUNT` та `SUM` для підрахунку загальної кількості фінансових цілей у системі та сумарної кількості грошей, яку користувачі планують накопичити.
* **Запит 1.2:** Використовує `AVG`, `MIN`, `MAX` разом із `GROUP BY` для обчислення середньої, найменшої та найбільшої транзакції в розрізі кожної категорії. Це допомагає зрозуміти масштаб витрат у різних категоріях.
* **Запит 1.3:** Використовує `COUNT` та `HAVING` для фільтрації згрупованих даних. Запит виводить лише ті фінансові рахунки, які є активними (мають більше однієї транзакції).
* **Запит 1.4:** Використовує `SUM` для підрахунку загальної суми бюджетних лімітів, встановлених кожним користувачем.

## 2. Об'єднання таблиць (JOIN)
Ці запити об'єднують дані з кількох нормалізованих таблиць для формування зрозумілих звітів:
* **Запит 2.1 (INNER JOIN):** Об'єднує таблиці `transactions`, `accounts` та `categories`. Замість незрозумілих ідентифікаторів (ID), запит виводить дату транзакції, реальну назву рахунку та назву категорії.
* **Запит 2.2 (LEFT JOIN):** Об'єднує `categories` та `transactions`. Використання `LEFT JOIN` гарантує, що в результаті з'являться всі наявні категорії, навіть ті, за якими користувачі ще не здійснювали жодних витрат чи доходів.
* **Запит 2.3 (RIGHT JOIN):** Об'єднує `transaction_tags` та `tags`. Дозволяє побачити всі існуючі теги в системі і транзакції, до яких вони прив'язані. `RIGHT JOIN` гарантує виведення тегів-сиріт, які ще не використовувалися.

## 3. Підзапити (Subqueries)
Підзапити дозволяють використовувати результати одного запиту як умову для іншого:
* **Запит 3.1 (Підзапит у WHERE):** Знаходить "аномальні" або просто великі витрати. Підзапит вираховує середню суму транзакції по всій базі, а основний запит фільтрує лише ті транзакції, які перевищують це значення.
* **Запит 3.2 (Підзапит у SELECT):** Для кожного користувача виконується підзапит, який сумує баланси всіх його фінансових рахунків, виводячи єдиний показник "загального капіталу" на рівні з іменем.
* **Запит 3.3 (Підзапит з IN):** Підзапит створює список `user_id`, які існують у таблиці `goals`, а основний запит виводить контактні дані лише тих користувачів, які збігаються з цим списком (тобто користувачів, які ставлять перед собою фінансові цілі).
